Module: common/components/sldsDatepicker

The slds-datepicker is a component to display and capture a date in a variety of formats. It uses Salesforece Lightning Design System styles and Moment.js for date operations.
Parameters:
Name Type Description
model String | Number | Date =? the selected date value. type depends on modelFormat
(Date | formatted String | timestamp Number/String)
isValid optional | Boolean =? true if selected model value is valid
(valid Date and between min / max if specified)
modelFormat optional | String @ date format used to load/save model value.
default/empty: Date | 'timestamp': timestamp in a string | YYYY-MM-DD
viewFormat optional | String @ date format used to display date (default: 'Do MMMM YYYY')
placeholder optional | String @ placeholder text shown when no date selected
locale optional | String @ user locale (by default, current browser locale)
min optional | String @ minimum allowed date (can be an offset in days)
max optional | String @ maximum allowed date (can be an offset in days)
Min and Max date can be specified as an absolute date (using "YYYY-MM-DD" format),
or using one of the keywords: "today", "now", "tomorrow", "yesterday", "next day", "last day",
"next week", "last week", "next month", "last month", "next year", "last year"
firstWeekDaySunday optional | Boolean =? true: first day column = Sunday | default: Monday
utc optional | Boolean =? true: format and select date with midnight UTC time | default: midnight current timezone
readOnly optional | Boolean =? true: make read-only | default: editable
required optional | Boolean < true: make input field value required | default: not required
disabled optional | Boolean = true: disable input field | default: enabled
disableClear optional | Boolean = true: when editable, disable clearing selected date | default: enabled if editable
keepOpen optional | Boolean =? true: keep the calendar popup open on selection | default: close on selection
hideErrors optional | Boolean =? true: hide validation error messages | default: show them
id optional | String @ id of this slds-datepicker element
inputId optional | String @ id of the text input containing the date view value
errorId optional | String @ id of the error message element
variable optional | String @ global variable name
position optional | String @ 'fixed' for use in a dialog (calculates position, makes calendar popup visible over all other elements), otherwise relative
onChange optional | function & function to call when selected date changes.

Examples
//Store model as Date object, between tomorrow and next week
<slds-datepicker variable="sd1" id="sdp1" model="vm.date1" 
first-week-day-sunday="false" placeholder="Pick a date SLDS" 
min="tomorrow" max="next week"></slds-datepicker>
//Store model as formatted string "YYYY-MM-DD"
//minimim allowed value: "2016-01-01"
<slds-datepicker variable="sd2" model="vm.date2" 
placeholder="Pick a date formatted" model-format="YYYY-MM-DD" view-format="MMMM Do, YYYY" 
min="2016-01-01"></slds-datepicker>
//Store model as numeric timestamp, position fixed
//allowed values: between "last month" and "next year" 
<slds-datepicker variable="sd3" model="vm.date3" first-week-day-sunday="true" 
placeholder="Pick a date timestamp" model-format="timestamp" 
min="last month" max="next year" position="fixed" keep-open="true"></slds-datepicker>
Dynamic properties as used in slds-table / edit-field
<slds-datepicker ng-if="!vm.isReadOnly() && vm.isDate()" 
model="vm.model[field]" view-format="{{vm.dateFormat}}" 
model-format="timestamp" placeholder="{{vm.properties.templateOptions.placeholder}}" 
position="fixed" min="{{vm.properties.min}}" max="{{vm.properties.max}}"></slds-datepicker>
