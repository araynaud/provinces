Module: common/components/editDialog

Displays a modal dialog box or popup panel that can contain: messages, buttons, editable fields, using a model object and different field types (field properties are like table columns).
Parameters:
Name Type Description
variable optional | String @ global variable name for this component controller instance
visible Boolean = controls visibility of the dialog/popup. true: open / default: closed
dialogTitle optional | String @ dialog header main title
dialogSubtitle optional | String = dialog header sub title
message optional | String @ text message to display in dialog content
content optional | String | jQuery @ jquery selector of page elements to move inside dialog content.
footerText optional | String = text variable to display on the left of dialog footer
readOnly optional | Boolean = make all dialog form fields read-only
size optional | String @ default: medium | large | small
position optional | String @ (popup-panel only) top, bottom, left, right | header | footer | sidebar
Some can be combined like: top left, bottom right, left sidebar
model optional | Object = object to display/edit
fields optional | Array.<Object> =? list of fields of the model to display/edit with their properties
fieldNames optional | Array.<String> @ field names of the model to display (converted to fields, for read-only dialog)
columns optional | Number =? Number of columns for the form in this dialog
actions Array.<object> = list of buttons displayed on right of the dialog footer.
Each action button can have theses properties:
text, title, click function, close, disable, show, showLoader, link | href
cancelButton optional | String @ default: empty: no Cancel button | text of cancel button ("No", "Close" , "Cancel", ...).
The Cancel button always does same action as the X close icon
(closes the dialog and calls the same onClose function if specified).
confirmButton optional | String @ default: empty: no Confirm button | text of cancel button ("Yes" , "OK", "Do it!" ...).
The Confirm button always closes the dialog and calls onConfirm then onClose functions if specified.
onConfirm optional | function = function executed when the Confirm button si clicked
onOpen optional | function = function executed when the dialog/popup opens
onClose optional | function = function executed when the dialog/popup closes
redirect optional | String = redirect after closing

Examples
Simple confirmation dialog (modal, centered) with 1 cancel button and 1 confirm button (OK)
//clicking on any of the buttons closes the dialog

<edit-dialog visible="vm.showDeleteDialog" size="medium"
dialog-title="Delete Container Template" dialog-subtitle="vm.formToDelete.Name"
message="Are you sure you want to delete this Container Template ?"
cancel-button="Cancel" on-close="vm.cancelDeleteForm"
confirm-button="OK"    on-confirm="vm.deleteForm">
</edit-dialog>
Modal dialog (modal, centered)
//Display and edit object vm.pdfDocument with list of fields defined in vm.pdfDialog.fields
//Shows Cancel and Save button spinning while converting.

//JS page controller
vm.pdfDialog = {
	visible: false,
	fields: [
		{field: "name",           type: "text", label: "Document Name", required: true },
		{field: "category",       type: "picklist", label: "Category",  required: true, options: categoryOptions },
		{field: "expirationDate", type: "date", label: "Expiration Date", min: "tomorrow" }
	],
	actions: [
		{ text: "Save", click: vm.convertToPDF, close: false, showLoader: function() { return vm.converting; } }
	];
};

//HTML
<edit-dialog variable="pdfSaveDialog" visible="vm.pdfDialog.visible"
dialog-title="Document Information" dialog-subtitle="'Save Form as PDF in Document Library'"
actions="vm.pdfDialog.actions" cancel-button="Cancel"
model="vm.pdfDocument" fields="vm.pdfDialog.fields"></edit-dialog>

<modal-dialog variable="pdfSaveDialog" visible="vm.pdfDialog.visible"
dialog-title="Document Information" dialog-subtitle="'Save Form as PDF in Document Library'"
actions="vm.pdfDialog.actions" cancel-button="Cancel"
model="vm.pdfDocument" fields="vm.pdfDialog.fields"></modal-dialog>
//Popup panel at footer of the page (non modal, can be positioned at top, bottom, left or right)

<popup-panel variable="pdfSaveDialog" visible="vm.pdfDialog.visible"
dialog-title="Document Information"
dialog-subtitle="'Save Form as PDF in Document Library'" position="footer"
actions="vm.pdfDialog.actions" cancel-button="Cancel"
model="vm.pdfDocument" fields="vm.pdfDialog.fields"></popup-panel>
